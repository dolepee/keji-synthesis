# KEJI Track Strategy

## Primary Track Map

### Protocol Labs: Let the Agent Cook

KEJI fits because it is an autonomous loop:

- discover or receive task
- plan
- pay
- execute
- verify
- log

Must-have deliverables:

- ERC-8004 registration tx
- `agent.json`
- `agent_log.json`
- real tool usage
- guardrails

### Protocol Labs: Agents With Receipts - ERC-8004

KEJI fits because identity and receipts are core, not decorative.

Must-have deliverables:

- ERC-8004 identity
- onchain verifiability
- manifest compatibility
- structured execution logs

### Bankr

KEJI fits if Bankr is the actual model-routing layer.

Must-have deliverables:

- Bankr-routed calls in the real execution path
- evidence of compute-aware operation
- clear role in the autonomous loop

### OpenServ

KEJI fits if we structure it as at least a planner and executor workflow.

Must-have deliverables:

- OpenServ is load-bearing
- workflow is useful, not decorative
- service outcome is real

### Merit Systems / AgentCash

KEJI fits if x402 payments are required to complete the task.

Must-have deliverables:

- paid request through AgentCash
- no mock payment flow
- x402 is part of the core loop

### Status Network

This is a qualifier, not the main story.

Must-have deliverables:

- verified contract deployment on Status Sepolia
- one gasless tx with proof
- AI agent component
- README or short demo clip

## Strategic Cuts

We are not chasing every sponsor.

Skipped because they increase risk more than upside:

- bond.credit
- Uniswap as core story
- Celo
- Lido
- Venice

## Judging Positioning

KEJI should be presented as:

"An autonomous agent CFO with identity, paid tool access, safety controls,
and verifiable receipts."

That framing works across the primary tracks without looking like sponsor glue.
